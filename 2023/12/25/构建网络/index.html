构建网络

Typora Markdown
PyTorch
Publish Date:   2023-12-25

神经网络由对数据执行操作的层/模块组成。torch.nn 命名空间提供了构建你自己的神经网络所需的所有基础组件。

PyTorch 中的每个模块都是 nn.Module 的子类。神经网络本身就是一个由其他模块（层）组成的模块。这种嵌套结构便于构建和管理复杂的架构。

1.获取用于训练的设备

import torch

# 检测 CUDA (NVIDIA GPU)
if torch.cuda.is_available():
    device = "cuda"
# 否则使用 CPU
else:
    device = "cpu"

print(f"Using {device} device")

2.定义神经网络

通过继承 nn.Module 来定义我们的神经网络，并在 __init__ 中初始化神经网络层。每个 nn.Module 子类都在 forward 方法中实现对输入数据的操作

import torch.nn as nn  # 导入PyTorch神经网络模块

# 继承自nn.Module的神经网络类
class NeuralNetwork(nn.Module):
    def __init__(self):
        super().__init__()  # 调用父类构造函数
        
        # 定义网络层结构 这里面nn.Flatten() 也可以加入nn.Sequential()含义一样
        self.flatten = nn.Flatten()  # 展平层：将二维图像压成一维向量
        
        # 线性层与激活函数的顺序组合
        self.linear_relu_stack = nn.Sequential(
            nn.Linear(28*28, 512),  # 全连接层1：784输入 → 512输出
            nn.ReLU(),              # 激活函数1：引入非线性
            nn.Linear(512, 512),    # 全连接层2：512 → 512
            nn.ReLU(),              # 激活函数2
            nn.Linear(512, 10),     # 输出层：512 → 10（对应10个类别）
        )

    def forward(self, x):
        # 前向传播过程
        x = self.flatten(x)             # [批大小, 1, 28, 28] → [批大小, 784]
        logits = self.linear_relu_stack(x)  # 通过多层处理
        return logits                    # 返回原始输出（未归一化）

2.1网络结构图解

输入 (图像) → Flatten → FC1(784→512) → ReLU → FC2(512→512) → ReLU → FC3(512→10) → 输出

2.2继承nn.Module

在PyTorch中，继承自nn.Module是构建神经网络的核心机制

2.2.1框架功能整合

nn.Module是PyTorch的基类，为所有神经网络组件提供统一接口

2.2.1.1自动参数跟踪

自动注册nn.Linear、nn.Conv2d等层的可学习参数（权重和偏置），便于优化器（如torch.optim.SGD）管理。

2.2.1.2计算图构建

通过前向传播（forward）自动构建动态计算图，支持自动微分（Autograd）

2.2.1.3设备管理

自动处理张量在CPU/GPU/MPS设备间的迁移（通过.to(device)）

2.2.3必要的方法重写

继承后必须实现两个核心方法：

2.2.3.1__init__：定义网络层（如全连接层、卷积层）

def __init__(self):
    super().__init__()  # 必须调用父类初始化
    self.layer = nn.Linear(10, 5)

2.2.3.2forward：定义数据如何通过网络层

def forward(self, x):
    return self.layer(x)  # 实际计算过程

2.2.4关键特性支持

通过继承获得以下功能：

特性 说明
参数访问 model.parameters()或者model.named_parameters()返回所有可训练参数（用于优化器）
模型保存/加载 支持torch.save(model.state_dict(), …)和model.load_state_dict()
模型结构可视化 通过print(model)或torchsummary库查看层结构
钩子函数（Hooks） 支持注册前向/反向传播的中间操作（如特征可视化）

class BrokenModel:

2.2.6super().__init__()

在 PyTorch 中，super().__init__() 是神经网络类初始化时必须调用的关键语句。它的作用是确保父类 nn.Module 的初始化逻辑正确执行，从而使模型能够正常工作

2.2.6.1基础概念

①super()：Python 内置函数，用于获取父类（超类）的引用

②__init__：Python 类的构造函数（初始化方法__

③super().__init__()：显式调用父类的构造函数

2.2.6.2 在 PyTorch 中的作用

当自定义的神经网络类继承自 nn.Module 时:

class MyModel(nn.Module):
    def __init__(self):
        super().__init__()  # ✅ 必须调用
        self.layer = nn.Linear(10, 5)

①触发父类初始化
调用 nn.Module 的 __init__() 方法，初始化父类的内部状态（例如注册参数的容器、设备管理逻辑等）

②启用自动参数跟踪
父类会初始化一个内部的 _modules 和 _parameters 字典，用于自动跟踪子层（如 nn.Linear）的参数

2.2.6.3不调用的后果

如果省略 super().__init__()： 

class BrokenModel(nn.Module):
    def __init__(self):
        # ❌ 未调用 super().__init__()
        self.layer = nn.Linear(10, 5)

model = BrokenModel()
print(list(model.parameters()))  # 输出空列表 []

①参数无法被识别
父类的初始化未执行，导致 nn.Linear 层的参数无法注册到模型中

②优化器失效
model.parameters() 返回空列表，优化器无法获取参数进行更新

③模型保存/加载失败
model.state_dict() 也无法正确记录参数。

2.2.6.4底层机制

nn.Module 的 __init__() 方法主要完成以下工作：

操作 说明
初始化 _parameters 字典 用于存储所有可学习参数（如 nn.Linear 的权重和偏置）
初始化 _modules 字典 用于存储子模块（如 nn.Sequential 中的层）
设置 training 属性 标记模型处于训练模式（model.train()）或评估模式（model.eval()）
初始化设备管理 记录张量所在的设备（CPU/GPU）

2.2.7代码示例（完整流程）

import torch.nn as nn

class NeuralNetwork(nn.Module):
    def __init__(self):
        super().__init__()
        self.stack = nn.Sequential(
            nn.Linear(28*28, 512),
            nn.ReLU(),
            nn.Linear(512, 10)
        )

    def forward(self, x):
        x = x.view(-1, 28*28)  # 展平输入
        return self.stack(x)

# 实例化并使用模型
model = NeuralNetwork()
print(model)  # 打印结构
X = torch.randn(64, 1, 28, 28)
output = model(X)  # 自动调用forward

2.3自动参数跟踪

在 PyTorch 中，自动参数跟踪是 nn.Module 类的核心功能之一，它解决了神经网络训练中一个关键问题：如何自动收集和管理模型的所有可训练参数（如权重、偏置）

2.3.1什么是可学习参数

神经网络中的可学习参数（Learnable Parameters）是模型通过训练数据自动调整的数值，例如：

①全连接层（nn.Linear）的权重矩阵（weight）和偏置向量（bias）

②卷积层（nn.Conv2d）的卷积核参数

③批归一化层（nn.BatchNorm2d）的缩放因子和偏移量

这些参数决定了模型的行为，训练过程就是通过反向传播不断优化这些参数的值。

2.3.2为什么要自动跟踪参数

假设没有自动参数跟踪，你需要手动完成以下操作：

​ ①在模型中定义参数（例如 self.weight = torch.randn(10, 5)）

​ ②手动将这些参数添加到一个列表中

​ ③将列表传递给优化器（如 optimizer = SGD([weight1, bias1, weight2, …], lr=0.1)）

问题：

​ ①容易遗漏参数（例如忘记某个层的偏置）

​ ②代码难以维护（尤其是深层网络）

​ ③无法利用 PyTorch 的模块化设计优势

2.3.3nn.Module 如何实现自动参数跟踪

当你继承 nn.Module 并定义网络层时，所有通过 PyTorch 预定义层（如 nn.Linear、nn.Conv2d）创建的参数会被自动注册到模型中。具体机制如下：

2.3.3.1步骤 1：定义层

import torch.nn as nn

class MyModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.linear = nn.Linear(10, 5)  # 输入10维，输出5维

2.3.3.2步骤 2：自动注册参数

nn.Linear 层内部已经定义了 weight 和 bias 参数

当通过 self.linear = nn.Linear(…) 将层赋值给模型属性时，nn.Module 会自动检测这些参数，并记录到 **model.parameters() **中

2.3.3.3步骤 3：优化器直接获取所有参数

model = MyModel()
optimizer = torch.optim.SGD(model.parameters(), lr=0.01)  # 一键获取全部参数

2.3.4验证参数跟踪

可以通过以下方式查看被自动跟踪的参数：

for name, param in model.named_parameters():
    print(f"参数名: {name}, 形状: {param.shape}")

# 输出示例：
# 参数名: linear.weight, 形状: torch.Size([5, 10])
# 参数名: linear.bias, 形状: torch.Size([5])

2.3.4.1model.named_parameters()

在 PyTorch 中，model.named_parameters() 是一个用于获取神经网络模型中所有可学习参数（带名称）的方法。它返回一个生成器（Generator），遍历时会生成每个参数的名称和参数对象本身，这对调试、参数操作和选择性训练非常有用。

2.3.4.1.1核心功能
2.3.4.1.1.1遍历所有可训练参数

返回模型中所有通过 nn.Module 定义的参数（例如 nn.Linear 的权重和偏置），包括嵌套子模块的参数。

2.3.4.1.1.2提供参数名称

参数名称的格式通常是：子模块名称.参数类型（例如 layer1.weight、block.conv.bias）。

2.3.4.1.1.3支持选择性操作

可以根据参数名称筛选特定层的参数（例如冻结某些层或修改特定参数）

2.3.4.1.2基本用法

示例代码：

import torch.nn as nn

# 定义一个简单模型
class MyModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.linear1 = nn.Linear(10, 5)  # 子模块名称: "linear1"
        self.linear2 = nn.Linear(5, 2)   # 子模块名称: "linear2"

    def forward(self, x):
        return self.linear2(self.linear1(x))

model = MyModel()

# 遍历所有参数（带名称）
for name, param in model.named_parameters():
    print(f"参数名称: {name}")
    print(f"参数形状: {param.shape}")
    print(f"参数值示例:\n{param.data}\n" + "-" * 50)

2.3.4.1.3 与 model.parameters() 的区别
方法 返回值 用途场景

2.3.5对比实验：手动管理参数的麻烦

如果不使用 nn.Module 的自动跟踪功能，需要这样写代码：

# ❌ 手动管理参数的繁琐方式
class ManualModel:
    def __init__(self):
        self.weight = torch.randn(5, 10, requires_grad=True)  # 手动定义参数
        self.bias = torch.zeros(5, requires_grad=True)

    def forward(self, x):
        return x @ self.weight.t() + self.bias

model = ManualModel()
optimizer = torch.optim.SGD([model.weight, model.bias], lr=0.01)  # 手动传递参数

缺点：

①每添加一个层，都要手动添加参数到列表

②容易遗漏参数（例如忘记某个层的 bias）

③无法支持复杂网络结构（如嵌套的子模块）

2.3.6自动参数跟踪的高级特性

2.3.6.1嵌套子模块的参数收集

如果一个模块包含其他子模块（例如 nn.Sequential），所有子模块的参数也会被自动收集：

class DeepModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.block = nn.Sequential(
            nn.Linear(10, 20),
            nn.ReLU(),
            nn.Linear(20, 5)
        )

model = DeepModel()
print(list(model.parameters()))  # 包含两个 Linear 层的 4 个参数（两个 weight 和两个 bias）

2.3.6.2参数冻结

可以通过设置 requires_grad=False 冻结某些参数（例如迁移学习时冻结预训练层）

for param in model.block[0].parameters():
    param.requires_grad = False  # 冻结第一层的参数

2.3.7自定义层的参数管理

如果想自定义一个层（例如实现一个特殊的全连接层），需要**显式使用 nn.Parameter **包装参数

class CustomLayer(nn.Module):
    def __init__(self):
        super().__init__()
        self.weight = nn.Parameter(torch.randn(5, 10))  # 必须用 nn.Parameter 包装
        self.bias = nn.Parameter(torch.zeros(5))

    def forward(self, x):
        return x @ self.weight.t() + self.bias

nn.Parameter 是 PyTorch 的特殊张量类型，它会自动被 nn.Module 识别为**可学习参数 **

如果直接使用普通张量（如 torch.Tensor），参数不会被注册到模型中

2.4创建一个 NeuralNetwork 的实例

创建一个 NeuralNetwork 的实例，并将其移动到 device ，然后打印其结构。

if torch.cuda.is_available():
    device = "cuda"
else:
    device = "cpu"

class NeuralNetwork(nn.Module):
    def __init__(self):
        super().__init__()
        self.flatten = nn.Flatten()
        self.linear_relu_stack = nn.Sequential(
            nn.Linear(28*28, 512),
            nn.ReLU(),
            nn.Linear(512, 512),
            nn.ReLU(),
            nn.Linear(512, 10),
        )

    def forward(self, x):
        x = self.flatten(x)
        logits = self.linear_relu_stack(x)
        return logits
 
model = NeuralNetwork().to(device)
print(model)

2.4.1model = NeuralNetwork().to(device)

不加 .to(device) 可以运行，但模型会默认在 CPU 上运行。

在PyTorch中，默认情况下，张量和模型都是在CPU上创建的。如果有可用的GPU，并且想要利用GPU加速训练，就需要将模型和张量都转移到GPU上。这通常是通过调用.to(device)来实现的，其中device是一个变量，可以是'cuda'（代表GPU）或者'cpu'。

如果仅需在 CPU 上运行（例如调试小规模数据），直接写 model = NeuralNetwork() 即可。

2.4.1.1如果需要 GPU 加速

若想利用 GPU 加速训练/推理，必须将模型显式转移到 GPU 设备（如 device = “cuda”）：

model = NeuralNetwork().to(device)  # 将模型参数和张量移到 GPU

①不加.to(device)时，模型默认在CPU上。

②如果数据和模型都在CPU上，代码可以运行，但速度较慢。

③如果数据在GPU而模型在CPU，或者反过来，会导致运行时错误。

④使用GPU加速需要显式转移模型和数据到同一设备。

2.4.1.2设备一致性要求

PyTorch 要求模型参数和输入数据必须在同一设备（CPU 或 GPU）上。例如：

正确代码：

device = "cuda" if torch.cuda.is_available() else "cpu"
model = NeuralNetwork().to(device)  # 模型在 GPU
x = torch.randn(64, 28, 28).to(device)  # 数据在 GPU
output = model(x)  # 正常计算

错误代码：

model = NeuralNetwork()  # 模型在 CPU
x = torch.randn(64, 28, 28).to("cuda")  # 数据在 GPU
output = model(x)  # 报错：Input type (CUDA) and weight type (CPU) mismatch

2.5运行实例

使用该模型，将输入数据传递给它。这会执行模型的 forward ，同时还会执行一些后台操作。请勿直接调用 model.forward() ！

在输入上调用模型会返回一个二维张量，其中维度 0 对应每个类别的 10 个原始预测值的每个输出，维度 1 对应每个输出的各个值。通过将其传递给 nn.Softmax 模块的一个实例来获得预测概率。

X = torch.rand(1, 28, 28, device=device)
logits = model(X)
pred_probab = nn.Softmax(dim=1)(logits)
y_pred = pred_probab.argmax(1)
print(f"Predicted class: {y_pred}")

2.6Model Layers 模型层

input_image = torch.rand(3,28,28)
print(input_image.size())		#输出：torch.Size([3, 28, 28])
flatten = nn.Flatten()
flat_image = flatten(input_image)
print(flat_image.size())		#输出：torch.Size([3, 784])
layer1 = nn.Linear(in_features=28*28, out_features=20)
hidden1 = layer1(flat_image)
print(hidden1.size())		#输出：torch.Size([3, 20])

2.6.1nn.Flatten

nn.Flatten默认会保留第一个维度（通常是批量大小）不变，然后将后面的所有维度展平。也就是说，如果输入的形状是(batch_size, dim1, dim2, …)，那么输出会是(batch_size, dim1dim2…)。这里的输入是(3,28,28)，假设这里的3是批量大小，那么展平后的结果应该是(3, 28*28)= (3,784)。所以输出确实是[3,784]。

2.6.2全连接层nn.Linear

全连接层（nn.Linear）的in_features指的是每个样本的特征数量，而不是整个批次的总体特征数。输入数据的形状通常是（batch_size, in_features），而全连接层会对每个样本独立地进行计算。因此，即使输入数据的形状是（3, 784），in_features仍然是784，因为每个样本有784个特征，而batch_size是3，这个维度会被自动处理。

2.6.3顺序容器nn.Sequential

nn.Sequential 是一个有序的模块容器。数据会按照定义的顺序依次通过所有模块。你可以使用顺序容器快速搭建一个像 seq_modules 这样的网络

seq_modules = nn.Sequential(
    flatten,
    layer1,
    nn.ReLU(),
    nn.Linear(20, 10)
)
input_image = torch.rand(3,28,28)
logits = seq_modules(input_image)

2.6.4nn.Softmax

神经网络的最后一个线性层返回对数几率（logits）——即取值范围在 $[-\infty, +\infty]$ 的原始值——这些值会被传递给 nn.Softmax 模块。对数几率会被缩放为 $[0, 1]$ 范围内的值，这些值代表了模型对每个类别的预测概率。dim 参数指定了需要将值求和为 1 的维度。

softmax = nn.Softmax(dim=1)
pred_probab = softmax(logits)

3.模型参数

神经网络内部的许多层都是可参数化的，即具有在训练期间进行优化的相关权重和偏差。对 nn.Module 进行子类化(定义模型是以nn.Module为父类，会自动跟踪你的模型对象内部定义的所有字段，并使所有参数可通过你的模型的 parameters() 或 named_parameters() 方法访问。
