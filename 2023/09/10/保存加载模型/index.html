保存加载模型

No tag
Publish Date:   2023-09-10

1.保存权重

import torch
import torchvision.models as models
model = models.vgg16(weights='IMAGENET1K_V1')
torch.save(model.state_dict(), 'model_weights.pth')

1.1model = models.vgg16(weights='IMAGENET1K_V1')

创建VGG16模型的实例，并加载在ImageNet数据集上预训练的权重

weights='IMAGENET1K_V1' 指定加载官方预训练的权重（在ImageNet-1k数据集上训练得到的版本1权重）

这会得到一个已具备图像分类能力的模型（输出1000个ImageNet类别）

1.2torch.save(model.state_dict(), 'model_weights.pth')

功能：将模型的权重参数保存到文件

  • model.state_dict() 返回一个包含模型所有参数（权重和偏置）的字典。
  • torch.save() 将这个字典序列化保存到名为model_weights.pth的文件中。
  • 保存的只是模型参数，不包含模型结构（后续加载时需要先实例化相同结构的模型）。

1.3预训练模型（Pre-trained Model）

  • 直接使用在大规模数据集（如ImageNet）上训练好的模型权重，省去从头训练的时间。
  • 适用于迁移学习（Transfer Learning），例如通过微调（Fine-tuning）适应新任务。

1.4state_dict

  • 在PyTorch中，state_dict是一个有序字典，将各层的名称映射到对应的参数张量。
  • 示例：{'features.0.weight': tensor(...), 'features.0.bias': tensor(...), ...}

1.5保存与加载的两种方式

1.5.1仅保存参数（推荐）

# 保存
torch.save(model.state_dict(), 'model_weights.pth')

# 加载
model = models.vgg16()  # 需要先创建相同结构的模型
model.load_state_dict(torch.load('model_weights.pth'))

1.5.2保存整个模型（包含结构，但灵活性差）：

torch.save(model, 'full_model.pth')
model = torch.load('full_model.pth')

1.5.3使用场景示例

假设希望在自己的图像分类任务中复用VGG16的特征提取能力：

①加载预训练权重初始化模型。

②替换最后一层全连接层（原输出1000类，改为想要的类别数）。

③保存调整后的模型权重供后续使用。

import torch
from torchvision import models

# 1. 加载预训练模型
model = models.vgg16(weights='IMAGENET1K_V1')

# 2. 修改最后一层（假设新任务有10个类别）
# 原来的预训练权重在除了最后一层之外的其他层都被保留下来，而最后一层的权重是随机初始化的。这是因为替换层的时候，新层的参数没有经过预训练，所以需要重新训练或者微调。
model.classifier[6] = torch.nn.Linear(4096, 10)

# 3. 保存新权重
torch.save(model.state_dict(), 'custom_vgg16_weights.pth')

1.6**weights_only=True**

1.6.1限制反序列化行为

在加载模型权重时，仅允许加载纯数据（如张量、数字、字符串等），禁止执行任何可能嵌入在文件中的代码或复杂对象。这是为了防止恶意代码通过模型文件传播。

1.6.1.1序列化与反序列化

  • 序列化（Serialization）：将程序中的对象（如神经网络权重、数据结构等）转换为二进制或文本格式，以便存储或传输。
    • 示例：torch.save(model.state_dict(), 'weights.pth') 将模型权重转换为二进制文件。
  • 反序列化（Deserialization）：将序列化后的数据重新转换为内存中的对象。
    • 示例：torch.load('weights.pth') 读取二进制文件并重建权重字典。

1.6.1.2默认行为的风险（weights_only=False）

PyTorch 默认使用 Python 的 pickle 模块实现序列化。pickle 的设计允许序列化任意 Python 对象（包括代码），这会导致以下风险：

恶意代码执行：如果攻击者篡改了模型文件（如 .pth 文件），可以在其中嵌入恶意代码，例如

# 恶意代码示例：通过 __reduce__ 方法注入代码
class MaliciousPayload:
    def __reduce__(self):
        import os
        return (os.system, ('rm -rf /',))  # 执行删除系统文件的命令
  • 当使用 torch.load() 加载此文件时，pickle 会直接执行 __reduce__ 中定义的代码，造成灾难性后果。

1.6.2**weights_only=True 的作用**

model = models.vgg16() # we do not specify ``weights``, i.e. create untrained model
model.load_state_dict(torch.load('model_weights.pth', weights_only=True))
model.eval()

设置 weights_only=True 后，PyTorch 会严格限制反序列化的内容：

  • 允许加载的数据类型：
    • 张量（torch.Tensor）
    • 基本数据类型（如数字、字符串、列表、字典等）
    • 特定受信任的 PyTorch 对象（如 Parameter、Module 的元数据）。
  • 禁止加载的内容：
    • 任何可能触发代码执行的 Python 对象（如函数、类实例、动态生成的代码）。
    • 所有通过 __reduce__、__setstate__ 等方法定义的代码逻辑。

1.6.3实际场景对比

场景 1：安全加载（weights_only=True）

# 假设文件内容仅包含张量（正常模型权重）
weights = torch.load('safe_weights.pth', weights_only=True)  # ✅ 安全
model.load_state_dict(weights)
  • 结果：仅加载张量数据，无任何代码执行。

场景 2：恶意文件攻击（weights_only=False）

# 假设文件被篡改，包含恶意代码
weights = torch.load('malicious_weights.pth')  # ❌ 默认 weights_only=False
  • 结果：触发嵌入的 __reduce__ 方法，执行恶意代码（如删除文件、窃取数据等）。

1.6.4注意事项

仅保存 state_dict：

  • 如果保存整个模型（torch.save(model, 'model.pth')），文件可能包含模型结构代码，此时 weights_only=True 可能导致加载失败。
  • 推荐做法：始终保存 state_dict（纯数据）而非完整模型。

2.保存和加载带有形状信息的模型

在加载模型权重时，需要先实例化模型类，因为该类定义了网络的结构。如果希望将这个类的结构与模型一起保存，在这种情况下，可以将 model （而不是 model.state_dict() ）传递给保存函数：

torch.save(model, 'model.pth')

保存 state_dict 被认为是最佳实践。不过，下面使用 weights_only=False ，因为这涉及加载模型，这是 torch.save 的一种传统用例。

model = torch.load('model.pth', weights_only=False)
